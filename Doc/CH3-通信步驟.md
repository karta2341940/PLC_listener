# 第3章 SLMP 通信步驟

外部設備與 SLMP 對應設備按照本章所示的步驟進行通信。

---

## 3.1 使用 TCP/IP 時

以 TCP/IP 進行 SLMP 通信時的通信步驟如下所示。

### 特性

| 項目 | TCP/IP |
|------|--------|
| 連接建立 | 通信前確立連接（三次握手） |
| 資料確認 | 確認資料正常傳送到通信物件 |
| 資料可靠性 | **高**（確保資料正確傳輸） |
| 線路負載 | **高**（相比 UDP/IP） |

### 通信步驟

```
外部設備                         SLMP對應設備
   │                                 │
   │──── ① 建立連接請求 ─────────────►│
   │◄─── ② 連接確認 ─────────────────│
   │                                 │
   │──── ③ 發送請求報文 ─────────────►│
   │◄─── ④ 接收回應報文 ────────────  │
   │                                 │
   │──── ⑤ 斷開連接 ────────────────►│
   │◄─── ⑥ 斷開確認 ─────────────────│
```

---

## 3.2 使用 UDP/IP 時

以 UDP/IP 進行 SLMP 通信時的通信步驟如下所示。

### 特性

| 項目 | UDP/IP |
|------|--------|
| 連接建立 | **無**（通信時未確立連接） |
| 資料確認 | **無**（不確認資料正常傳送到通信物件） |
| 資料可靠性 | **低**（相比 TCP/IP） |
| 線路負載 | **低**（相比 TCP/IP） |

### 通信步驟

```
外部設備                         SLMP對應設備
   │                                  │
   │──── ① 發送請求報文 ─────────────►│（無連接確立）
   │◄─── ② 接收回應報文 ─────────────│（無傳輸確認）
```

---

## 3.3 注意事項

### 請求報文的發送

在從外部設備發送請求報文之前，應**確認 SLMP 對應設備處於可接收請求報文的狀態**。

### 發送多個請求報文的情況下

應在外部設備中對請求報文的**副幀頭設置序號**並進行發送。  
通過設置序號，在發送了多個請求報文的情況下，可在外部設備側**判別回應報文的發送源**。

> 詳細參照：第4章「副幀頭」說明

### 連續發送請求報文的情況下

在不等待回應報文接收的狀況下連續發送設置了序號的請求報文的情況下，外部設備應避免超出下表所示的**陳述式數限制**：

| 產品名稱 | 型號 | 1個連接中可處理的陳述式數 |
|---------|------|----------------------|
| CC-Link IE 現場網路乙太網路適配器模組 | NZ2GF-ETB | 1＋（50 ÷ 使用的連接數） |
| 乙太網路搭載模組 | RJ71EN71 | 1＋（接收緩衝可存儲報文數(190) ÷ 使用的連接數） |
| 乙太網路搭載模組 | QJ71E71-100、LJ71E71-100 | 1＋（57 ÷ 使用的連接數） |
| Q系列 CPU（UDVCPU/UDPVCPU 等） | Q03/Q04/Q06/Q13/Q26 系列 | 1＋（接收緩衝可存儲報文數(576) ÷ 使用的連接數） |
| RCPU | — | 1＋（接收緩衝可存儲報文數(32) ÷ 使用的連接數） |
| LHCPU | — | 1＋（接收緩衝可存儲報文數(32) ÷ 使用的連接數） |

> ⚠️ 計算結果為小數的情況下，將捨去小數點後置為整數。

超出陳述式數時，SLMP 對應設備中有可能發生出錯，或有可能未從 SLMP 對應設備返回回應報文。  
此時應**降低發送請求報文的頻率**。

### 未返回回應報文的情況下

通過 SLMP 對應設備未返回回應報文的情況下，應在經過請求報文的**「監視計時器」**中設置的時間後，通過外部設備**再次發送請求報文**。

### 關於 SLMP 對應設備的更換

故障等導致更換外部設備及 SLMP 對應設備時，由於連接設備的 **MAC 地址發生變化**，因此有可能無法進行通信（更換為相同 IP 地址的設備的情況下）。  
更換了乙太網路內的設備的情況下，應**重新啟動網路內的全部設備**。

### 訪問 CPU 模組的情況下

通過外部設備經由 SLMP 對應設備訪問 CPU 模組時的注意事項：

#### CPU 模組側的處理時機

**在 CPU 模組的 END 處理時**，執行對於請求報文的處理。

```
外部設備                         SLMP對應設備           CPU 模組
   │                                  │                    │
   │──── 1. 發送請求報文 ────────────►│                    │
   │                                  │──── 2. 讀取/寫入請求 ─►│
   │                                  │      (END 處理時執行)   │
   │                                  │◄─── 3. 返回處理結果 ────│
   │◄─── 4. 返回回應報文 ─────────────│                    │
```

**處理流程說明：**

1. 通過外部設備將請求報文發送到 SLMP 對應設備中
2. SLMP 對應設備接收請求報文後，按照請求內容對 CPU 模組進行資料讀取或寫入請求
3. CPU 模組在程式的 **END 處理**時按照外部設備請求的內容進行資料的讀取或寫入，將處理結果返回到 SLMP 對應設備中
4. SLMP 對應設備在受理處理結果時，將包括處理結果的回應報文發送至外部設備中

#### CPU 模組為 RUN 中的讀取或寫入

- 由於對來自外部設備的請求進行處理，**CPU 模組的掃描時間將會延長**，應加以注意
- 掃描時間的延長影響控制的情況下，應以**較少的點數分數次**進行訪問
- 進行寫入的情況下，應確認 CPU 模組側**已允許 RUN 中寫入**（是否解除系統保護等）

#### 對訪問目標的 CPU 模組添加系統保護時

在訪問目標中將發生出錯，且異常回應被返回到外部設備中。  
應在**解除 CPU 模組側的系統保護**後，再次發送請求報文。

#### 多個外部設備對同一站同時發生了訪問請求時

根據請求時機，進行多次 END 處理之前，來自外部設備的請求有可能被等待。  
通過下述方法之一，可以在 1 個掃描以內處理多個請求：

- 通過程式執行 `COM` 陳述式
- 使用工程工具的「服務處理設置」，將服務處理時間確保為 **1～100ms**
