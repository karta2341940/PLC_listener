# 第5章 陳述式

SLMP 的陳述式及子陳述式設置到請求資料中。本章說明所有可用陳述式的格式、子陳述式、請求資料、回應資料及通信示例。

---

| 子陳述式 | 位數 | 示例（輸入 X） |

| 子陳述式 | 資料容量 | 示例（輸入 X） |
|---------|---------|-------------|
| 003 / 002 | 2位元組 | `9CH 00H` |
| 001 / 000 | 1位元組 | `9CH` |

### 起始軟元件編號（軟元件編號）

指定進行讀取或寫入的軟元件編號，連續軟元件指定起始編號。根據軟元件類型以**10進制數**或**16進制數**指定。

**ASCII 代碼（子陳述式 003/002：8位；子陳述式 001/000：6位）**

| 子陳述式 | 位數 | 示例（編號 1234） |
|---------|------|----------------|
| 003 / 002 | 8位 | `00001234` → `30H 30H 30H 30H 31H 32H 33H 34H` |
| 001 / 000 | 6位 | `001234` → `30H 30H 31H 32H 33H 34H` |

**二進位碼（子陳述式 003/002：4位元組；子陳述式 001/000：3位元組）**

| 子陳述式 | 資料容量 | 示例（M1234） |
|---------|---------|------------|
| 003 / 002 | 4位元組 | `D2H 04H 00H 00H`（10進制→16進制：1234→0x4D2） |
| 001 / 000 | 3位元組 | `D2H 04H 00H` |

---

### 讀取資料、寫入資料格式

#### 位元單位（子陳述式 001、003）

**ASCII 代碼**：從起始軟元件按順序發送，ON→`31H`（"1"），OFF→`30H`（"0"）

**二進位碼**：以4位指定1點，從起始軟元件按順序發送

**示例（M10～M14 的 5點 ON/OFF，M10=ON, M11=OFF, M12=ON, M13=OFF, M14=ON）：**

| 格式 | 資料表示 |
|------|---------|
| ASCII | `10101` → `31H 30H 31H 30H 31H` |
| 二進位 | `10H 10H 10H`（每4位1點，高低位組合） |

#### 字單位（子陳述式 000、002）

**ASCII 代碼**：將1字以4位元（16進制）表示，從高位開始發送

**二進位碼**：以16位（2位元組）表示1字，存儲順序從低位元組到高位元組

### 5.2.1 Read（陳述式：0401）

**功能**：通過軟元件讀取值

#### 請求資料

| 欄位 | ASCII代碼 | 二進位碼 |
|------|----------|---------|
| 陳述式 | `0401` → `30H 34H 30H 31H` | `01H 04H` |
| 子陳述式（位元單位） | `0001` 或 `0003` | `01H 00H` 或 `03H 00H` |
| 子陳述式（字單位） | `0000` 或 `0002` | `00H 00H` 或 `02H 00H` |
| 軟元件代碼 | 4位或2位 ASCII | 2位元組或1位元組 |
| 起始軟元件編號 | 8位或6位 ASCII（16進制） | 4位元組或3位元組 |
| 軟元件點數 | 4位 ASCII（16進制） | 2位元組 |

**可指定軟元件點數：**

| 讀取方式 | ASCII代碼 | 二進位碼 |
|---------|---------|---------|
| 位元單位 | 1～3584點 | 1～7168點 |
| 字單位 | 1～960點 | 1～960點 |

> ⚠️ 不可指定長計時器觸點（LTS）、線圈（LTC）、長累計計時器觸點（LSTS）、線圈（LSTC）、長變址寄存器（LZ）

#### 通信示例（以位元單位讀取 M100～M107）

**ASCII 代碼：**
```
請求資料: 0401 0001 M* 000100 0008
          ↓    ↓    ↓  ↓      ↓
          陳述式 子陳述式 軟元件代碼 起始編號 點數
回應資料: 00010011  ← M100=OFF,M101=OFF,M102=ON,M103=OFF,...
```

**二進位碼：**
```
請求資料: 01H 04H 01H 00H 64H 00H 00H 90H 08H 00H
          ↓         ↓        ↓              ↓        ↓
          陳述式      子陳述式   起始編號(100)   代碼    點數
回應資料: 00H 01H 00H 11H
```

## 5.3 Label（標籤訪問）

### 前提條件

- **不可以**訪問局部標籤、模組標籤
- **不可以**訪問通過 GX Works2 設置的全域標籤
- **不可以**訪問安全 CPU 的安全全域標籤、安全局部標籤、一般/安全共用標籤
- 需要通過 GX Works3 的全域標籤設置編輯器將**「來自於外部設備的訪問」設置為有效**（默認為無效）
- 以 ASCII 代碼進行資料通信時，需要從 UTF-16 將標籤名轉換為 ASCII 代碼，報文容量增加

### 5.3.1 Array Label Read（陳述式：041A）

**功能**：從排列型標籤及結構體的構件為排列的標籤中讀取資料

| 子陳述式 | ASCII | 二進位 |
|---------|-------|-------|
| — | `0000` | `00H 00H` |

**請求資料包含：**
- 陳述式（`041A`）

## 5.6 Remote Control（遠端操作）

### 遠端操作前注意事項

- 遠端操作後，若進行訪問目標電源 OFF→ON 或復位，**遠端操作資訊將被刪除**
- CPU 模組開關為 RUN 狀態下進行遠端 STOP，復位 CPU 模組時將**變為 RUN 狀態**
- 對訪問目標 CPU 模組施加**系統保護**時，不可以通過外部設備進行遠端操作
- 建議使用 **UDP 協議**進行遠端操作（TCP 在復位時連接被切斷，需要重新確立連接）
- 通過1次陳述式**僅可對1站**進行遠端操作

---

### 5.6.1 Remote Run（陳述式：1001）

**功能**：對訪問目標模組執行遠端 RUN

> ⚠️ 遠端 RUN 可以在訪問目標模組開關為 RUN 時使用。開關為 STOP 的情況下雖然正常完成，但**訪問目標不變為 RUN 狀態**。

#### 請求資料

| 欄位 | ASCII代碼 | 二進位碼 |
|------|----------|---------|
| 陳述式 | `1001` → `31H 30H 30H 31H` | `01H 10H` |
| 子陳述式 | `0000` | `00H 00H` |
| 模式 | `0001`（不強制）/`0003`（強制） | `01H 00H`/`03H 00H` |
| 清除模式 | `00`（不清除）/`01`（清除鎖存外）/`02`（清除全部） | `00H`/`01H`/`02H` |
| 保留 | `00` | `00H` |

**模式說明：**

| 模式值 | 說明 |
|--------|------|
| `0001` | 不強制執行（其它設備遠端 STOP/PAUSE 過程中不進行遠端 RUN） |
| `0003` | 強制執行（即使其它設備遠端 STOP/PAUSE 過程中也進行遠端 RUN） |

**清除模式說明：**

| 清除模式值 | 說明 |
|-----------|------|
| `00` | 不清除軟元件 |
| `01` | 清除鎖存範圍外的軟元件 |
| `02` | 清除包括鎖存範圍在內的全部軟元件 |

**回應資料**：無

**通信示例（不強制執行、清除全部軟元件）：**
```
ASCII:  1001 0000 0001 02 00
二進位: 01H 10H 00H 00H 01H 00H 02H 00H
```

## 5.8 File（檔案控制）

File 陳述式在外部設備通過 CPU 模組讀取/寫入參數及程式等儲存檔案時使用。

### 重要注意事項

- 關於通過 SLMP 可訪問的檔案，請參閱所使用的 CPU 模組的用戶手冊
- 有可能看到系統用檔案（為系統用，**請勿訪問**）

### 子陳述式區別

| 子陳述式 | 適用模組 | 口令長度 |
|---------|---------|---------|
| `0000` | QCPU、LCPU 等 | 4字元固定長 |
| `0004` | QCPU（部分功能） | 4字元固定長 |
| `0040` | RCPU（MELSEC iQ-R 系列） | 6～32字元可變長 |

### 共用欄位說明

#### 口令

| 子陳述式 | 格式 | 說明 |
|---------|------|------|
| `0000`/`0004` | 4字元固定長（ASCII/二進位皆用 ASCII）| 未設置口令時填空格（`20H`） |
| `0040` | 字元數（`0000H`～）+ 口令（6～32字元）| 未設置口令時字元數指定 `0`，不指定口令 |

> ⚠️ 如果連續一定次數輸入錯誤口令，將**閉鎖**，一定時間內無法解除口令。

#### 驅動器 No.

| 驅動器 | 說明 |
|--------|------|
| `0` | 標準 ROM（程式記憶體） |
| `1` | 記憶卡（ROM） |
| `2` | 記憶卡（RAM） |
| `3` | 標準 RAM |
| `4` | RCPU 用 SD 卡 |

#### 檔案名

- 以 UTF-16 編碼的檔案名
- ASCII 代碼通信時：UTF-16 數值轉為 ASCII 代碼（每字元4位 ASCII 表示），從高位到低位發送
- 二進位碼通信時：UTF-16 數值直接發送，從低位到高位發送

#### 屬性

| 屬性值 | 說明 |
|--------|------|
| `0000` | 讀取/寫入（可讀可寫） |
| `0001` | 讀取專用 |

#### 檔案指針 No.

- 在 Open File（`1827`）打開檔案時獲取
- 後續 Read File、Write File、Close File 操作使用此編號

### 執行步驟

#### 新建資料型檔案

```
1. 確認檔案有無       → Read Directory/File (1810) 或 Search Directory/File (1811)
2. 檔案名登錄與容量確保 → New File (1820)
3. 檔案打開（鎖定）    → Open File (1827)
4. 寫入檔案內容       → Write File (1829)
5. 檔案關閉（解鎖）    → Close File (182A)
6. 確認檔案 No.       → Search Directory/File (1811)（僅 MELSEC-Q/L 系列需要）
```

#### 覆蓋現有檔案

```
1. 確認檔案 No.       → Search Directory/File (1811)
2. 更改屬性（解除唯讀） → Change File State (1825)
3. 檔案打開（鎖定）    → Open File (1827)
4. 寫入檔案內容       → Write File (1829)
5. 檔案關閉（解鎖）    → Close File (182A)
```

#### 刪除檔案

```
1. 確認檔案 No.       → Search Directory/File (1811)（Q/L 系列）
2. 更改屬性（解除唯讀） → Change File State (1825)（唯讀的情況下）
3. 刪除檔案           → Delete File (1822)
```

**回應資料**：無

## 5.9 Self Test（回送測試）（陳述式：0619）

**功能**：對外部設備與乙太網路搭載模組的通信是否正常動作進行測試

> ⚠️ **僅對與外部設備連接的乙太網路搭載模組**可以進行回送測試。對於經由網路的其它站模組不可以使用。

| 欄位 | ASCII代碼 | 二進位碼 |
|------|----------|---------|
| 陳述式 | `0619` → `30H 36H 31H 39H` | `19H 06H` |
| 子陳述式 | `0000` | `00H 00H` |
| 回送資料數 | 4位 ASCII（16進制，1～960位元組） | 2位元組 |
| 回送資料 | 最大960字元半形字元串（"0"～"9"、"A"～"F"） | 最大960位元組 |

**回應資料**：與請求報文中指定的「回送資料數」和「回送資料」**相同內容**

**通信示例（回送資料為 "ABCDE"）：**
```
ASCII 請求:  0619 0000 0005 ABCDE
ASCII 回應:  0005 ABCDE

二進位請求: 19H 06H 00H 00H 05H 00H 41H 42H 43H 44H 45H
二進位回應: 05H 00H 41H 42H 43H 44H 45H
```
