# 第4章 報文格式

本章對 SLMP 的報文格式進行說明。

---

## 4.1 請求報文

通過外部設備發送到 SLMP 對應設備的**請求報文**格式如下所示。

### 請求報文結構

#### 3E幀 / 4E幀 請求報文

```
┌────────┬────────┬──────────────────────────────────────────────────────┬────────┐
│ 幀頭   │ 副幀頭 │            請求資料部                                │ 頁腳   │
│(Frame  │(Sub-   ├─────────┬──────────┬────────┬──────┬────────┬───────┤(Footer)│
│ Header)│Header) │請求目標 │請求目標  │請求目標│請求目│請求資  │陳述式 │        │
│        │        │網路No.  │站號      │模組I/O │標多點│料長    │資料   │        │
│        │        │(1 byte) │(1 byte)  │編號    │站號  │(2 byte)│       │        │
│        │        │         │          │(2 byte)│(1byte│        │       │        │
└────────┴────────┴─────────┴──────────┴────────┴──────┴────────┴───────┴────────┘
```

**固定為 0 的項目：**
- 固定值：`00H`
- 系統區域：`0000000000H`

---

### 各欄位說明

#### 幀頭（Frame Header）

- TCP/IP 及 UDP/IP 用的幀頭
- 在外部設備側進行添加及發送（通常由外部設備自動添加）

---

#### 副幀頭（Sub-Header）

由以下兩部分組成：
- **固定值**：表示框架類型及請求報文（3E幀僅為固定值）
- **序號區域**：報文判別用的任意編號（`0000H`～`FFFFH`），3E幀無法設置序號

**序號說明：**  
與請求報文相同的序號也被存儲到回應報文中，因此在發送多個請求報文的情況下，可在外部設備側**判別回應報文的發送源**。

**各框架類型的副幀頭格式（以序號 `1234H` 為例）：**

| 框架類型 | ASCII 格式 | 二進位格式 |
|----------|-----------|-----------|
| **3E幀** | `5000` → `35H 30H 30H 30H` | `50H 00H` |
| **4E幀** | `540012340000` → `35H 34H 30H 30H 31H 32H 33H 34H 30H 30H 30H 30H` | `54H 00H 34H 12H 00H 00H` |
| **站號擴展幀** | — | `68H 00H 34H 12H 00H 00H` |

**序號注意事項：**
- 序號應在外部設備側管理並使用
- 以 **ASCII 代碼**發送時：按照從**高位位元組到低位元位元組**的順序存儲
- 以**二進位碼**發送時：按照從**低位元位元組到高位位元組**的順序存儲

---

#### 請求目標網路 No.、請求目標站號

指定訪問目標的**網路 No.** 和**站號**，以 16 進制數進行指定。  
按照從**高位位元組到低位元位元組**的順序進行發送。

| 訪問目標 | 請求目標網路 No. | 請求目標站號 |
|---------|---------------|------------|
| A（連接站/本站） | `00H` | `FFH` |
| B（其它站） | `01H`～`EFH`（1～239）：網路 No. | `01H`～`78H`（1～120）：站號 |
| B（其它站，站號 ≥ 121） | 同上 | `7CH`（124）：在站號擴展幀的請求目標擴展站號區域中指定 |
| 管理站/主站（參數設置） | 同上 | `7DH`（125） |
| 當前管理站/主站 | 同上 | `7EH`（126） |

> ⚠️ 對於網路 No. 為 240～255 的站，**不可以訪問**。

**示例（請求目標網路 No. 為 `1AH`）：**

| 格式 | 資料 |
|------|------|
| ASCII | `1A` → `31H 41H` |
| 二進位 | `1AH` |

---

#### 請求目標模組 I/O 編號

指定訪問目標的模組：

| 訪問目標 | 請求目標模組 I/O 編號 |
|---------|---------------------|
| CPU 模組（本站/管理CPU） | `03FFH` |
| 多CPU 1號機 | `03E0H` |
| 多CPU 2號機 | `03E1H` |
| 多CPU 3號機 | `03E2H` |
| 多CPU 4號機 | `03E3H` |
| 控制系統 CPU | `03D0H` |
| 待機系統 CPU | `03D1H` |
| A系統 CPU | `03D2H` |
| B系統 CPU | `03D3H` |
| 多點連接站 CPU | `0000H`～`01FFH` |
| CC-Link IE 現場網路遠端起始模組（本站） | `03FFH` |
| CC-Link IE 現場網路遠端起始模組 1號機 | `03E0H` |
| CC-Link IE 現場網路遠端起始模組 2號機 | `03E1H` |

**執行 File（檔案控制）陳述式時的指定規則：**

| 訪問目標 | 請求目標模組 I/O 編號 |
|---------|---------------------|
| CPU 模組（本站） | `03FFH` |
| A系統 CPU | `03D2H` |
| B系統 CPU | `03D3H` |
| CC-Link IE 現場網路遠端起始模組（本站） | `03FFH` |
| CC-Link IE 現場網路遠端起始模組 1號機 | `03E0H` |
| CC-Link IE 現場網路遠端起始模組 2號機 | `03E1H` |

> ⚠️ 在執行 File 陳述式的過程中系統切換時，訪問目標被切換，因此**不可以**指定 `03D0H`（控制系統）或 `03D1H`（待機系統）。

**資料表示方式：**
- ASCII 代碼：按照從高位位元組到低位元位元組的順序進行發送
- 二進位碼：按照從低位元位元組到高位位元組的順序進行發送

---

#### 請求目標多點站號

連接站（本站）的訪問目標為多點連接站的情況下進行指定：

| 連接站的訪問目標 | 請求目標多點站號 |
|----------------|---------------|
| B（多點連接站） | `00H`～`1FH`（0～31）：站號 |
| A（對多點連接與網路進行中繼的站） | `00H` |
| 不是多點連接站的站 | `00H` |

---

#### 請求目標擴展站號（僅站號擴展幀）

在固定值 `00`（1位元組）的區域後面，以 16 進制數指定訪問目標的站號（121及以上），共 **2 位元組**。

| 想要指定的站號 | 請求目標站號區域設置 | 請求目標擴展站號區域設置 |
|-------------|---------------------|----------------------|
| 1～120 | `01H`～`78H`（直接填站號） | `0000H`（0） |
| 121～65534 | `7CH`（124） | `0001H`～`FFFEH`（實際站號） |
| 管理站/主站 | `7CH`（124） | `0000H`（0） |
| 本站 | — | `FFFFH`（65535） |

---

#### 請求資料長

以 16 進制數指定從**監視計時器開始到請求資料為止**的資料長（單位：位元組）。

**示例（請求資料長為 24 位元組 = 18H）：**

| 格式 | 資料 |
|------|------|
| ASCII | `0018` → `30H 30H 31H 38H` |
| 二進位 | `18H 00H` |

---

#### 監視計時器

對 SLMP 對應設備向訪問目標請求處理後，到返回回應為止的**等待時間**進行設置的計時器：

| 設置值 | 含義 |
|--------|------|
| `0000H`（0） | 無限等待（在處理完成之前繼續等待） |
| `0001H`～`FFFFH`（1～65535） | 等待時間（**單位：250ms**） |

**建議設置範圍：**

| 訪問目標 | 監視計時器建議值 |
|---------|---------------|
| 本站 | `01H`～`28H`（0.25秒～10秒） |
| 其它站 | `02H`～`F0H`（0.5秒～60秒） |

**示例（監視計時器設置為 `10H`）：**

| 格式 | 資料 |
|------|------|
| ASCII | `0010` → `30H 30H 31H 30H` |
| 二進位 | `10H 00H` |

---

#### 請求資料

指定表示請求內容的**陳述式**、**子陳述式**、**資料**。  
（詳細參照第5章「陳述式」）

站號擴展幀的請求資料格式：`陳述式 + 子陳述式 + 固定值00（1位元組）+ 系統區域（5位元組）+ 資料`

---

#### 頁腳（Footer）

- TCP/IP 及 UDP/IP 用的頁腳
- 在外部設備側進行添加及發送（通常由外部設備自動添加）

---

## 4.2 回應報文

通過 SLMP 對應設備發送到外部設備的**回應報文**格式如下所示。

### 回應報文結構

#### 3E幀 / 4E幀 回應報文

```
┌────────┬────────┬──────────────────────────────────────────────────────┬────────┐
│ 幀頭   │ 副幀頭 │             回應資料部                               │ 頁腳   │
│(Frame  │(Sub-   ├─────────┬──────────┬────────┬──────┬────────┬───────┤(Footer)│
│ Header)│Header) │請求目標 │請求目標  │請求目標│請求目│回應資  │結束   │        │
│        │        │網路No.  │站號      │模組I/O │標多點│料長    │代碼   │        │
│        │        │(1 byte) │(1 byte)  │編號    │站號  │(2 byte)│+資料  │        │
│        │        │         │          │(2 byte)│(1byte│        │       │        │
└────────┴────────┴─────────┴──────────┴────────┴──────┴────────┴───────┴────────┘
```

**存儲與請求報文相同內容的項目：**
- 請求目標網路 No.
- 請求目標站號
- 請求目標模組 I/O 編號
- 請求目標多點站號
- 陳述式
- 子陳述式
- 請求目標擴展站號（僅站號擴展幀）

---

### 各欄位說明

#### 幀頭

存儲乙太網路的幀頭。

---

#### 副幀頭

存儲表示框架類型及**回應報文的固定值**，以及與請求報文相同的序號（3E幀僅為固定值）。

**各框架類型的副幀頭格式（以序號 `1234H` 為例）：**

| 框架類型 | ASCII 格式 | 二進位格式 |
|----------|-----------|-----------|
| **3E幀** | `D000` → `44H 30H 30H 30H` | `D0H 00H` |
| **4E幀** | `D40012340000` → `44H 34H 30H 30H 31H 32H 33H 34H 30H 30H 30H 30H` | `D4H 00H 34H 12H 00H 00H` |
| **站號擴展幀** | — | `E8H 00H 34H 12H 00H 00H` |

**序號存儲順序：**
- ASCII 代碼：按照從**高位位元組到低位元位元組**的順序存儲
- 二進位碼：按照從**低位元位元組到高位位元組**的順序存儲

---

#### 回應資料長

以 16 進制數存儲從**結束代碼開始到回應資料為止**的資料長（單位：位元組）。

**示例（回應資料長為 22 位元組 = 16H）：**

| 格式 | 資料 |
|------|------|
| ASCII | `0016` → `30H 30H 31H 36H` |
| 二進位 | `16H 00H` |

---

#### 結束代碼

存儲陳述式**處理結果**：

| 結果 | 結束代碼 | 說明 |
|------|---------|------|
| 正常結束 | `0000H`（0） | — |
| 異常結束 | 訪問目標的出錯代碼 | 例：`0400H` |

**正常結束時的格式示例：**

| 格式 | 資料 |
|------|------|
| ASCII | `0000` → `30H 30H 30H 30H` |
| 二進位 | `00H 00H` |

**異常結束時（出錯代碼 `0400H`）的格式示例：**

| 格式 | 資料 |
|------|------|
| ASCII | `0400` → `30H 34H 30H 30H` |
| 二進位 | `00H 04H` |

---

#### 回應資料（3E幀或4E幀）

**正常結束時：**  
存儲對於陳述式的讀取資料等。

**異常結束時，存儲以下內容：**
- 出錯回應站的資訊（網路 No.、站號、請求目標模組 I/O 編號、多點站號）
- 與請求報文相同的陳述式及子陳述式
- 異常結束時的回應資料（通過陳述式定義的情況下）

> ⚠️ 出錯回應站的資訊為**進行了出錯回應的站**的資訊，因此可能與請求報文的內容不相同。

---

#### 回應資料（站號擴展幀）

**正常結束時：**  
存儲與請求報文相同的陳述式及子陳述式、固定值 `00`（1位元組）、系統區域（5位元組）、對於陳述式的讀取資料等。

**異常結束時，存儲以下內容：**
- 與請求報文相同的陳述式及子陳述式
- 固定值 `00`（1位元組）
- 系統區域（5位元組）
- 出錯回應站的資訊（網路 No.、站號、請求目標模組 I/O 編號、多點站號、固定值 `00`（1位元組）、擴展站號）
- 異常結束時的回應資料（通過陳述式定義的情況下）
