## 附1 通過軟元件的擴展指定進行讀取、寫入

通過將請求資料內的子陳述式置為 `008`，可以進行以下訪問：

- 至連結直接軟元件的訪問
- 至模組訪問軟元件的訪問
- 至 CPU 緩衝記憶體訪問軟元件的訪問
- 通過變址寄存器間接指定網路 No. 及起始輸入輸出編號的訪問
- 通過變址寄存器或長變址寄存器間接指定軟元件編號的訪問
- 通過字軟元件中存儲的值間接指定軟元件編號的訪問

### 可使用的陳述式

| 類型 | 操作 | 陳述式 |
|------|------|--------|
| Device | Read | `0401` |
| Device | Write | `1401` |
| Device | Read Random | `0403` |
| Device | Write Random | `1402` |
| Device | Entry Monitor Device | `0801` |
| Device | Read Block | `0406` |
| Device | Write Block | `1406` |

### 子陳述式（擴展指定用）

| 訪問方式 | ASCII代碼 | 二進位碼 |
|---------|---------|---------|
| 以位元單位訪問 | `0081` | `81H 00H` |
| 以位元單位訪問 | `0083` | `83H 00H` |
| 以字單位訪問 | `0080` | `80H 00H` |
| 以字單位訪問 | `0082` | `82H 00H` |

---

### 附1.1 至連結直接軟元件的訪問

訪問遠端輸入（RX）、遠端輸出（RY）、連結特殊繼電器（SB）等的網路模組的連結軟元件。

#### 擴展指定的軟元件代碼

| 軟元件 | 類型 | ASCII（iQ-R，4位） | ASCII（Q/L，2位） | 二進位 | 編號進制 |
|--------|------|------------------|----------------|-------|---------|
| 連結輸入（X） | 位 | `X***` | `X*` | `009CH`/`9CH` | 16 進制 |
| 連結輸出（Y） | 位 | `Y***` | `Y*` | `009DH`/`9DH` | 16 進制 |
| 連結繼電器（B） | 位 | `B***` | `B*` | `00A0H`/`A0H` | 16 進制 |
| 連結特殊繼電器（SB） | 位 | `SB**` | `SB` | `00A1H`/`A1H` | 16 進制 |
| 連結寄存器（W） | 字 | `W***` | `W*` | `00B4H`/`B4H` | 16 進制 |
| 連結特殊寄存器（SW） | 字 | `SW**` | `SW` | `00B5H`/`B5H` | 16 進制 |

#### 擴展指定格式（ASCII 代碼）

```
報文格式（子陳述式 0081/0080）：
[子陳述式][擴展指定][起始軟元件或軟元件編號]...
J[網路No.(3位ASCII)]
```

**擴展指定（網路 No.）的格式：**

| 格式 | 說明 |
|------|------|
| ASCII | 以16進制數（3位 ASCII）指定網路 No.，如 `J008` 表示網路 No.=8 |
| 二進位 | 以16進制數（2位元組）指定網路 No.，如 `08H 00H` |

**通信示例（訪問網路 No.1 的 W100，即 J1\W100）：**

```
ASCII 請求資料:
0080 00 J001 W* 000100000

二進位請求資料:
80H 00H 00H 00H 00H 01H 00H B4H 00H 00H 01H 00H F9H
```

---

### 附1.2 至模組訪問軟元件的訪問

訪問 SLMP 對應設備及智慧功能模組的緩衝記憶體。

#### 格式說明

模組訪問軟元件的表示格式：`U[模組No.]\G[緩衝記憶體地址]`

| 格式 | 說明 |
|------|------|
| ASCII | `U` + 模組No.（ASCII）+ `\G` + 緩衝記憶體地址 |
| 二進位 | 以 `F8H` 標識，後跟模組No. 和地址 |

---

### 附1.3 至 CPU 緩衝記憶體訪問軟元件的訪問

訪問 CPU 模組的緩衝記憶體。

格式：`U3E[CPU No.]\G[地址]`

---

### 附1.4 通過變址寄存器間接指定網路 No. 及起始輸入輸出編號的訪問

使用 CPU 模組的**變址寄存器（Z）**，可以間接指定訪問物件的網路 No.。

**擴展指定（使用變址寄存器）格式：**

| 格式 | 說明 |
|------|------|
| ASCII | `J` + 網路No.（3位）+ 變址寄存器（2位），如 `J001Z0` |
| 二進位 | 以變址寄存器標識位加入地址 |

**通信示例（訪問網路 No.1+Z0 的 W100，即 J1+Z0\W100）：**

```
ASCII 請求資料:
0080 00 J001Z0 W* 000100000

二進位請求資料:
80H 00H 00H 00H 00H 01H 00H B4H 00H 40H 01H 00H F9H
```

---

### 附1.5 通過變址寄存器或長變址寄存器間接指定軟元件編號的訪問

使用 CPU 模組的**變址寄存器（Z）**或**長變址寄存器（LZ）**，可以間接指定訪問物件的軟元件編號。

**適用條件：**
- 子陳述式 `008`（擴展指定）
- 在起始軟元件編號中加入變址寄存器指定

---

### 附1.6 通過字軟元件中存儲的值間接指定軟元件編號的訪問

使用**字軟元件（D 等）中存儲的值**，可以間接指定訪問物件的軟元件編號。

## 附3 訪問目標為多 CPU 系統的情況下

本章對訪問目標為**多 CPU 系統**時的 SLMP 通信進行說明。

> 關於多 CPU 系統，請參閱所使用的 CPU 模組手冊（多 CPU 系統篇）。

### 訪問範圍

可以對訪問目標的**管理 CPU** 及**非管理 CPU** 進行訪問。

### 可訪問陳述式

| 類型 | 操作 |
|------|------|
| Device | Read（`0401`）、Write（`1401`）、Read Random（`0403`）、Write Random（`1402`） |
| Device | Entry Monitor Device（`0801`）、Execute Monitor（`0802`） |
| Device | Read Block（`0406`）、Write Block（`1406`） |
| Label | Array Label Read（`041A`）、Array Label Write（`141A`） |
| Label | Label Read Random（`041C`）、Label Write Random（`141B`） |
| Extend Unit | Read（`0601`）、Write（`1601`） |
| Remote Control | Remote Run（`1001`）、Remote Stop（`1002`）、Remote Pause（`1003`） |
| Remote Control | Remote Latch Clear（`1005`）、Remote Reset（`1006`）、Read Type Name（`0101`） |
| File | Read Directory/File（`1810`）、Search Directory/File（`1811`）、New File（`1820`） |
| File | Delete File（`1822`）、Copy File（`1824`）、Change File State（`1825`） |
| File | Change File Date（`1826`）、Open File（`1827`）、Read File（`1828`）、Write File（`1829`）、Close File（`182A`） |
| Self Test | Self Test（`0619`） |
| Clear Error | Clear Error（`1617`） |

### 請求目標模組 I/O 編號的指定（多 CPU 系統）

| 訪問目標 | 請求目標模組 I/O 編號 |
|---------|---------------------|
| 管理 CPU | `03FFH` |
| 多CPU 1號機 | `03E0H` |
| 多CPU 2號機 | `03E1H` |
| 多CPU 3號機 | `03E2H` |
| 多CPU 4號機 | `03E3H` |
