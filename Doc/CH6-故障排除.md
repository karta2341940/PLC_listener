# 第6章 故障排除

本章對通過外部設備無法與 SLMP 對應設備進行通信時，外部設備側的原因確定及處理方法進行說明。

> 關於 **SLMP 對應設備側**的故障排除，請參閱所使用的 SLMP 對應設備手冊。

---

## 故障排除檢查表

| 檢查項目 | 處理方法 |
|---------|---------|
| **使用 TCP/IP 時，是否確立與 SLMP 對應設備的連接？** | 應通過外部設備對 SLMP 對應設備進行連接請求（Active 打開） |
| **是否通過外部設備發送請求報文？** | 應通過外部設備對 SLMP 對應設備發送請求報文；應確認請求報文的發送目標是否為支援 SLMP 的設備 |
| **網路的線路負載是否過高？** | 應降低通過外部設備發送請求報文的頻率；應減少網路的線路負載 |
| **IP 地址是否正確？** | 外部設備的 IP 地址的網路部分應與 SLMP 對應設備一致；請勿使其它乙太網路設備與 IP 地址重複；應將請求報文的目標 IP 地址置為 SLMP 對應設備的 IP 地址 |
| **協議（TCP/IP 或 UDP/IP）是否正確？** | 外部設備的協議應與 SLMP 對應設備中設置的協議一致 |
| **埠編號是否正確？** | 應將請求報文的目標埠編號與 SLMP 對應設備中設置的本站埠編號一致 |
| **通信資料代碼（ASCII 代碼或二進位碼）在外部設備與 SLMP 對應設備中是否一致？** | 應將請求報文的通信資料代碼與 SLMP 對應設備中設置的通信資料代碼一致 |
| **請求報文的格式是否正確？** | 應以本手冊中記載的報文格式發送請求報文（參照第4章「報文格式」） |
| **請求報文內指定的資料的存儲順序及值的範圍是否正確？** | 應在本手冊記載的存儲順序及範圍內進行指定（參照第4章「報文格式」及第5章「陳述式」） |
| **回應報文的「結束代碼」是否變為 0？** | 「結束代碼」為 0 以外的情況下，在 SLMP 對應設備將發生出錯；應通過所使用的 SLMP 對應設備手冊確認結束代碼後進行處理 |
| **使用 TCP/IP 時，外部設備中預定接收的回應報文長與實際接收的回應報文長是否一致？** | 實際回應報文長**短於**預定的情況下，應進行接收殘餘資料的處理；實際回應報文長**長於**預定的情況下，應確認請求報文，並復位 SLMP 對應設備 |
| **是否設置防火牆？** | 應重新審核防火牆的設置 |
| **使用標籤訪問的情況下，是否通過 GX Works3 的全域標籤設置編輯器將「來自於外部設備的訪問」設置為有效？** | 應通過 GX Works3 的全域標籤設置編輯器將「來自於外部設備的訪問」的設置專案設置為**有效** |

---

## 故障排除流程圖

```
通信失敗
    │
    ▼
確認連接建立（TCP/IP）？
    ├─ 否 → 執行 Active 打開，建立連接
    └─ 是 ▼
    
確認請求報文是否發送？
    ├─ 否 → 發送請求報文
    └─ 是 ▼

確認 IP 地址、埠編號、協議設置
    ├─ 不符 → 修正設置
    └─ 符合 ▼

確認通信資料代碼（ASCII/二進位）
    ├─ 不符 → 修正代碼設置
    └─ 符合 ▼

確認報文格式（第4章）
    ├─ 錯誤 → 修正報文格式
    └─ 正確 ▼

確認回應報文的結束代碼
    ├─ 非 0 → 查閱 SLMP 設備手冊的出錯代碼說明
    └─ 為 0 ▼

確認防火牆設置
    ├─ 有問題 → 重新審核防火牆設置
    └─ 正常 ▼

確認 GX Works3 標籤設置（使用標籤訪問時）
```

---

## 常見出錯代碼對應措施

> 出錯代碼的詳細說明請參閱所使用的 **SLMP 對應設備手冊**。

| 結束代碼 | 常見原因 |
|---------|---------|
| `0000H` | 正常完成 |
| `0400H` | 陳述式/子陳述式不支援或設備側錯誤 |
| 非 `0000H` 的值 | 訪問目標發生出錯 → 查閱設備手冊 |
